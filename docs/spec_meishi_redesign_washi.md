# 名刺デザイン再設計：和紙・木札スタイル（本格版）

## 背景・経緯

直前の修正（letter-spacing・墨帯・X位置・CMYK金色調整）はレイアウトの微調整に過ぎず、
デザインとしての質は上がっていないという指摘を受けた。現状の
`print-card/assets/card-bg.png` は単純なCSS的グラデーション＋横線の繰り返しで、
質感が薄く「テンプレート感」が強い。今回は本格的に作り直す。

方向性は「和紙・木札スタイル」。サイト本体（`style.css` の `.cast-card`）が
実際の木目写真 `images/wood-texture.jpg` を使い、キャスト名を彫刻風に見せる
処理（`text-shadow`によるエンボス）をしていることを踏襲し、名刺にも同じ
デザイン言語を持ち込んで一貫性を出す。

`index.html` / `style.css` / `script.js` / `secret-card.html` /
`card-generator.js` など、`print-card/` 以外のファイルには一切手を加えないこと。
`images/wood-texture.jpg` は読み込み専用の参照元として使い、変更しないこと。

## 全体アーキテクチャ方針（重要）

これまでの実装は「背景画像はほぼ固定のまま、CSSとReportLabそれぞれで
テキストや帯を重ねる」方式だったため、RGB版とCMYK版で見た目がズレるリスクが
あった。今回は、**背景・フレーム・組子模様・木札プレート（名前の背景）を
すべて1本のPythonアセット生成スクリプトで新しい背景画像として一度に描き込み**、
RGB版（`assets/card-bg.png`）とCMYK版（`assets/card-bg-logo-cmyk.jpg`）の
2つを出力する。card.css / build.py 側は、その上にテキストを乗せるだけの
薄い実装にする。これにより2パスの見た目のズレを構造的に防ぐ。

新規スクリプト例: `print-card/generate_background.py`
（`build.py` から呼び出すか、`build.py` 冒頭で自動実行してよい。
Pillow は既存依存。必要なら `numpy` を requirements.txt に追加してよい
— タイル状のテクスチャ合成・グラデーション生成に有用）

出力先・サイズ:
- `print-card/assets/card-bg.png`（RGB、97mm×61mm相当の高解像度。
  最低 2200×1387px 以上。既存の1337×841pxより大幅に高精細化すること）
- `print-card/assets/card-bg-logo-cmyk.jpg`（同内容をCMYKで書き出し）

既存の `card.html` / `build.py` の `drawImage` 呼び出しはファイル名を
変えずに済むよう、同じファイル名で上書きしてよい（内容だけ差し替え）。

## デザイン仕様

### 1. 背景ベース：本物の木目テクスチャ
- `images/wood-texture.jpg`（812×542px、実写の木目写真）を読み込み、
  ミラータイリング（鏡映反転を組み合わせて継ぎ目を目立たなくする手法）で
  出力解像度まで拡大・敷き詰める。単純な引き伸ばしはぼやけるので避けること。
- 色調をブランドカラー（`--wood-brown: #5c4033` 〜 `--cedar: #8b6f47`）に
  寄せるため、暗めの茶色でカラーグレーディング（乗算 or オーバーレイブレンド）する。
  実写木目の明るいハニー色そのままではブランドと合わないので必ず調色すること。
- 周辺を暗く・中央〜左寄りをやや明るくするビネットを軽くかける
  （既存のグラデーション感を、写真ベースでより自然に再現するイメージ）。

### 2. 組子模様（麻の葉）のオーバーレイ
- 二重金枠の内側全体に、麻の葉文様（60度グリッドの六角形状の線模様）を
  金色 `#d4af37` の細線（線幅0.15mm相当）・不透明度 8〜12% で薄く重ねる。
  線間隔の目安は7〜9mm。
- 背景の一要素として溶け込ませ、文字の可読性を絶対に妨げないこと
  （後段でテキスト背後に追加のコントラスト対策があるため、模様自体は
  控えめでよい）。

### 3. 金枠を「箔押し風グラデーション」に
- 現状はベタ塗りの二重線。これを、線に沿って明暗が滑らかに変化する
  グラデーションストローク（例: 暗めの金→ハイライト→暗めの金、を
  枠に沿って複数箇所に配置し、光の反射のような表現にする）に変更する。
- 加えて、二重枠の内側4隅（安全エリアの外・枠のすぐ内側）に、
  小さな菱形（菱紋、一辺4〜5mm程度）の金アクセントを1つずつ配置する。
  主張しすぎない大きさ・不透明度にすること。

### 4. 名前エリアを「黒塗り木札」に置き換え
現状の `rgba(26,26,26,0.32)` の単純な半透明矩形をやめ、以下に置き換える。

- 背景生成スクリプト側で、個人名の表示位置（目安: 幅42mm×高さ13mm、
  現行の座標付近）に、`images/wood-texture.jpg` を暗く・彩度を落として
  焼いたような色調（黒に近い濃茶。既存の `tools/image-converter.html` の
  「焼き木フィルター」= grayscale/sepia/brightness/contrast の考え方を
  参考にしてよい）に加工した木札プレートを描き込む。
- プレートの上下端に細い金の縁取り（0.3mm程度）を入れ、実際に木札が
  掛けられているような印象にする。
- プレート自体は背景画像に焼き込み、`card.css` / `build.py` 側は
  その上にテキストのみを描画する（矩形の描画コードは削除してよい）。

### 5. 名前テキストのエンボス表現
`.name`（個人名 "yuki__san"）に、サイト本体の `.cast-name` が使っている
`text-shadow` によるエンボス表現を参考にした処理を加える。

- `card.css`側: `.name` に
  `text-shadow: 0 1px 1px rgba(0, 0, 0, 0.6), 0 -1px 0 rgba(255, 255, 255, 0.12);`
  程度を追加し、彫り込まれたような陰影を付ける
- `build.py`側（ReportLab）: `text-shadow` は無いため、同じ文字を
  1px相当オフセットした暗い色→さらに逆オフセットした薄い色→本体の
  順で複数回 `drawString` して疑似エンボスを作る

## セルフチェック（必須）

- 生成した `card-bg.png` / `card-bg-logo-cmyk.jpg` を開いて、木目の継ぎ目が
  目立たないこと、色調がブランドカラーに合っていることを目視確認する
- 麻の葉模様が濃すぎて文字やQRの可読性を落としていないか確認する
- 金枠のグラデーションが不自然なムラ・バンディングになっていないか確認する
- 木札プレートが安全エリア（中央85mm×49mm）内に収まっていること
- 名前テキストが木札プレートに対して十分なコントラストを保っていること
- 団体名・副題・X表記・QRなど、これまで実装済みの要素の位置関係は崩さないこと
  （今回のスコープは背景・フレーム・名前プレートの質の向上のみ）
- 最終的に `python build.py` を実行し、以下を再確認する
  - PDFページサイズが97×61mm
  - CMYK PDF内画像が `/DeviceCMYK`
  - QRが `https://mesukemo.uk` として正しくスキャン・デコードできる
  - プレビューPNGはPDFに埋め込まれたフォント・画像を正しく描画できるツールで
    生成すること（この環境にGhostscriptが無い場合はPyMuPDF等を使う。
    フォールバックで代替フォントになるレンダラは使わないこと）
