# 会員カードジェネレーターへの実QRコード実装仕様書

## 背景・ゴール

`secret-card.html` / `card-generator.js` の会員カードには、`drawCardFooter()` 関数の
コメントに「QR + モットー」とあるが、実際には QR コードは描画されておらず、
右側には装飾用のひし形ドット（`ctx.rotate` で回転させた小さい四角の連続）が
描かれているだけだった（コメントと実装が食い違っている状態）。

このひし形ドットを、実際にスキャン可能な QR コードに置き換える。
QR のリンク先は `https://mesukemo.uk`（サイトTOP）で固定する。

`index.html` / `style.css` / `script.js` / `print-card/` など、他のファイルには
一切手を加えないこと。

## 変更対象

- `secret-card.html`
- `card-generator.js`
- 新規追加: QR行列生成用の依存なしバニラJSライブラリ（後述）

## 技術方針

- 外部CDN・ネットワーク依存は禁止。VRChatイベント当日オフライン環境でも
  動作する必要があるため、QRエンコードライブラリはリポジトリ内に静的ファイルとして
  vendor し、`<script>` タグで読み込むこと（ビルドツール・npm不要、既存プロジェクトの
  構成方針と同じ）。
- 推奨ライブラリ: `qrcode-generator`（kazuhikoarase 氏、MITライセンス、単一ファイル・
  依存ゼロ）。`qr.isDark(row, col)` でモジュール単位の暗/明が取得できるAPIを使う。
  同等の依存ゼロ・MITライセンスの実装であれば代替可。
- 取得したQR行列は `<img>` や `data:` URL 画像として貼るのではなく、
  `ctx.fillRect()` でモジュール単位に直接描画すること。
  理由: `drawCard(ctx, scale)` は `ctx.scale(scale, scale)` を一度適用してから
  論理座標（`CARD_W`×`CARD_H` = 1024×650）で描画しており、ダウンロード時は
  `scale` を上げて（2x/3xなど）再描画している。ラスター画像で貼ると高解像度
  ダウンロード時にQRがぼやけてスキャン不能になりうるため、ベクター的に
  fillRect で描くことで常にシャープに保つ。
- ライブラリファイルは `vendor/qrcode-generator.min.js`（新規ディレクトリ）に配置し、
  `secret-card.html` で `card-generator.js` より前に `<script>` 読み込みする。

## 配置・レイアウト

`drawCardFooter(ctx, theme)` 内、現在ひし形ドットを描画している箇所
（`CARD_W - 140` 〜 `CARD_W - 28` の範囲、`cy` 中心）を置き換える。

- QRコードのモジュール描画エリア: 一辺 約92px（論理px、CARD_W=1024基準）
- 周囲に明るい背景の台座（角丸矩形、色は `#f5f3ed`（paper-white）または
  `#ebe6d9`（cream））を敷く。台座のパディングは片側8px程度
  （＝台座全体は約108×108px）。QRのクワイエットゾーンとして機能させる。
- 台座を含めた配置: 右端は既存ドットと同じ右余白を保つ（右端から約28px内側）。
  垂直方向は `cy`（ストリップ中央）を中心に配置。
- QRモジュール色は台座に対して常に高コントラストな暗色固定でよい
  （例: `#1a1a1a` ink-black や `#2d2d2d` charcoal）。テーマ（regular / cast / vip）
  によって色を変える必要はない。台座を敷くことで、カード全体がどのテーマ・
  どの背景色でも常にスキャン可能な状態を保証する。
- 台座とQRを追加した結果、モットーテキスト（中央）やロゴ（左）の位置・サイズを
  詰める必要が出た場合は、既存要素が窮屈にならない範囲で最小限調整してよい。
  ただし文字が欠けたり重なったりしないこと。

## 動作要件

- `cardType`（regular / cast / vip）のどれを選んでも、QRが正しく生成・描画されること
- プレビュー（scale=1）とダウンロード（scale=2, 3など `downloadScale` 経由）の
  両方でQRの見た目が崩れないこと（`drawCard(ctx, scale)` 経由で確認）
- QRの内容は固定文字列 `https://mesukemo.uk`
- エラー訂正レベルは M 以上を推奨（多少の汚れ・低解像度でも読み取れるように）

## セルフチェック

実装後、以下を確認すること。

- ブラウザで `secret-card.html` を開き、3種類のテーマそれぞれでカードを生成し、
  QRが台座付きで正しく表示されることを目視確認する
- 可能であれば、生成したPNG（ダウンロード機能経由）をPythonの `pyzbar` または
  `opencv` の `QRCodeDetector` でデコードし、`https://mesukemo.uk` が正しく
  読み取れることを確認する（該当ライブラリが環境にない場合はインストールして検証してよい）
- `drawCardFooter` のコメント「ボトムストリップ：QR + モットー」が実装と一致した
  状態になっていることを確認する
