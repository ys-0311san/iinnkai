# 名刺ジェネレーター 第2段: キャンバスでの位置調整（ドラッグ移動）機能

## 背景

`print-card/generator/`（前回実装済み）は、写真・セリフ・名前・Xアカウントの4項目を
フォーム入力するとCMYK印刷用PDFをダウンロードできるFlaskツール。現状は全要素が
固定位置で自動配置されるため、アップロードする写真によっては顔とセリフが重なる等、
見た目のバランスが悪くなることがある（実際に確認済み）。

これを解消するため、送信前にブラウザ上のキャンバスプレビューで
**写真・名前・セリフの位置をドラッグで調整できる**ようにする。

参考実装: プロジェクトルートの `card-generator.js`（既存の会員カードジェネレーター）に、
写真をキャンバス上でドラッグ移動する処理が実装済み（`photoX`/`photoY`変数、
`mousedown`/`mousemove`/`mouseup`/`mouseleave` と `touchstart`/`touchmove`/`touchend`両対応、
キャンバスの内部解像度と表示サイズの比率を `scaleX`/`scaleY` で補正）。
このパターンをそのまま参考にして実装すること（コードの丸コピーではなく、
このツール用のキャンバス構成に合わせて書き直してよい）。

## 変更対象

`print-card/generator/` 配下のみ。`print-card/generate_background.py` /
`print-card/build.py` は今回も一切変更しない。

## 要件

### 1. キャンバスプレビュー

- `templates/index.html` に61mm×97mmの名刺表面をプレビューするcanvas要素を追加する
  （表示解像度は任意でよいが、アスペクト比 61:97 を厳守し、物理mm座標とcanvas
  ピクセル座標の変換係数を明確にコード上に持つこと。例:
  `PX_PER_MM = canvasWidth / 61`）
- 写真をファイル選択した時点で、その画像をキャンバス上にフルブリードで敷いて
  プレビュー表示する（サーバー側の`cover_image`と近い見た目になるよう、
  JS側でも同様のcover配置＋ドラッグオフセットを実装する）
- 名前・セリフの入力欄にテキストを打つと、キャンバス上にリアルタイムで
  そのテキストのプレビュー（白文字＋黒縁取り、名前は右下・横書き、セリフは
  左寄り・縦書き風に数文字ごと改行）が重ねて表示される
- 丸ロゴ・Xアカウント表記・安全エリアの目安線（薄いグレーの枠線でよい）も
  プレビューに表示し、実際の印刷結果がイメージできるようにする

### 2. ドラッグ移動

以下の3種類の要素をそれぞれドラッグで移動できるようにする。

- **写真（背景）**: `card-generator.js`と同様の実装。マウス・タッチ両対応。
  ドラッグ範囲の制限は基本的に設けない（フルブリード素材なので、多少はみ出す
  移動が自然。ただし画像が完全に無くなる位置までは動かせないよう、
  最低限どこかは必ず61×97mmの範囲を覆っているように制限してよい）
- **名前テキスト**: ドラッグでキャンバス上の任意位置に移動できる
- **セリフテキスト**: ドラッグでキャンバス上の任意位置に移動できる（縦書き風の
  複数列テキストは、まとめて1つのブロックとして一緒に移動する。個々の文字を
  バラバラに動かす必要はない）

実装上、3つの要素は重なる可能性があるため、クリック/タップした座標が
どの要素の上かを判定し、一番上（zオーダー的に名前・セリフが写真より手前）の
要素から優先的にドラッグ対象として掴む処理にすること。

### 3. テキスト要素の移動範囲制限（最重要）

**名前テキストとセリフテキストは、ページ端から5mm以内の領域に入れないようにする。**
内訳は「断裁のズレ塗り足し3mm＋印刷のズレ許容2mm＝計5mm」で、これはユーザー向けに
説明する必要はなく、単に「5mm」という定数として扱ってよい。

- 具体的には、テキスト要素のバウンディングボックス（文字の描画範囲）が
  ページ座標系で `x < 5mm`、`x > 56mm`（61-5）、`y < 5mm`、`y > 92mm`（97-5）の
  いずれの領域にもはみ出さないよう、ドラッグ中の座標をクランプ（制限）すること
- **写真（背景）はこの制限の対象外。** 写真はページ全体（0〜61mm, 0〜97mm）の
  どこにでも自由に動かせる（塗り足し部分まで含めて動かせるのが正しい挙動）
- クランプは「動かした先でその位置を超えられない」形で実装する
  （例: `clamp(x, 5mm, 61mm - 5mm - textWidth)` のようにテキストの幅・高さを
  考慮した上下限を計算し、ドラッグ中の座標をその範囲に丸める）
- 制限にぶつかった際、視覚的にわかるよう軽くハイライト（枠の色を変える等）
  してもよいが必須ではない

### 4. サーバーへの送信

- フォーム送信時、以下をhidden inputとして追加で送信する:
  - `photo_offset_x_mm`, `photo_offset_y_mm`（写真の左上を基準にしたmm単位オフセット。
    どちらが原点かはJS側の実装に合わせて統一し、`card_builder.py`側でも
    同じ基準で解釈できるようにドキュメントコメントを残す）
  - `name_x_mm`, `name_y_mm`（名前テキストの基準点のmm座標）
  - `catchphrase_x_mm`, `catchphrase_y_mm`（セリフブロックの基準点のmm座標）
- 未ドラッグ（デフォルトのまま）の場合は、現状の固定デフォルト位置と
  同じ値をhidden inputの初期値として埋めておくこと（JSでキャンバス初期化時に
  デフォルト座標を設定し、それをそのままhidden inputの初期値にする）

### 5. `card_builder.py` 側の対応

- `build_custom_front()` と `_draw_front_signature()` に、位置を受け取る
  オプション引数を追加する（例: `photo_offset: tuple[float, float] | None = None`,
  `name_pos_mm: tuple[float, float] | None = None`,
  `catchphrase_pos_mm: tuple[float, float] | None = None`。Noneの場合は
  現行の固定デフォルト値にフォールバックする）
- **サーバー側でも5mmの安全マージン制限を再検証すること。**
  クライアント側のJSでクランプしていても、フォームは直接POSTでも送れてしまうため、
  `generate_pdf()` 内で受け取った `name_pos_mm` / `catchphrase_pos_mm` が
  5mm境界を超えていないか必ずチェックし、超えていれば5mm境界にクランプしてから
  描画する（写真オフセットはこの検証対象外でよい）
- `app.py` の `/generate` ルートで、上記の新しいhiddenフィールドを読み取り、
  mm単位のfloatとしてパースして `generate_pdf()` に渡す。パース失敗時は
  デフォルト値（None＝現行の固定位置）にフォールバックする

## セルフチェック

- ブラウザで `http://localhost:5000/` を開き、写真をアップロードすると
  キャンバスにフルブリードでプレビュー表示されること
- 名前・セリフを入力すると、キャンバス上にリアルタイムでテキストプレビューが
  重なって表示されること
- 写真をドラッグすると自由に移動し、キャンバス外にはみ出せること
- 名前・セリフをドラッグすると移動するが、ページ端から5mm以内には
  絶対に入り込まないこと（実際にドラッグして端に寄せてみて、5mm手前で
  止まることを目視確認する）
- 送信すると、キャンバスで見た通りの位置に写真・名前・セリフが配置された
  CMYK PDFがダウンロードされること
- hidden inputを消したり不正な値を送っても（例: curlで直接POST）、
  サーバー側で5mm制限が適用され、テキストが安全エリア外に出力されないこと
- 既存の `python build.py`（確定済みyuki__san名刺）に影響がないこと
  （`generate_background.py` / `build.py` が無変更であること）
