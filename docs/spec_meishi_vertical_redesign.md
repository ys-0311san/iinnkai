# 名刺を縦型に変更し、表面を一枚絵ベースのデザインに作り替える

## 背景

現在 `print-card/` は横型（仕上がり91mm×55mm、塗り足し込み97mm×61mm）の
両面名刺。これを **縦型**（仕上がり55mm×91mm、塗り足し込み61mm×97mm）に変更する。
併せて表面のデザインを、現行の「木札プレート＋別トリミングのキャラ写真＋小さいロゴ」の
組み合わせ構成から、**一枚のイラストをフルブリード背景にして、その上に名前・
既存コピー・団体ロゴだけを重ねるポスター的な構成**に作り替える。

`print-card/` 以外のファイルには一切手を加えないこと。

## 印刷サイズの変更（最重要・寸法を全面差し替え）

- 仕上がりサイズ: **55mm × 91mm**（縦向き）※現行の 91mm×55mm から縦横入れ替え
- 塗り足し: 各辺3mm → ページサイズ **61mm × 97mm**
- 安全エリア: 各辺6mm内側 → **49mm × 85mm**
- `print-card/generate_background.py` の `PAGE_MM` と `print-card/build.py` の
  `PAGE_MM` / `FINISH_MM` / `SAFE_MM` を上記の値に差し替える
- `card.css` の `@page`・`html,body`・`.card` の width/height も
  `61mm × 97mm` に差し替える
- README（`print-card/README.md`）に「アクセアの発注フォームでは
  “縦型名刺” のオプションを選んで入稿すること」という注意書きを追記する

## 表面（front）の新デザイン

### 元素材

以下のファイルを表面の背景素材として使用する。リポジトリ外のパスにあるため、
`print-card/assets/front-photo-source.png` としてコピーしてから加工すること。

```
/mnt/e/picture2/ee381eef14057097.png  (1200×1600px, RGB)
```

この画像には次の3つの要素が焼き込まれている。
1. キャラクターイラスト（画像中央〜右寄り）
2. コピー文言「手で解け、口で蕩ける。」（画像左側、縦書き風に配置。白文字＋縁取り）
3. 画像最下部（下端から画像高さの約8%＝下から約130px程度の帯）に、
   ピンク地の団体プロモーションバナー（「メスケモ好きなら今すぐ入ろう」＋
   丸いマスコット＋「メスケモ推進委員会」の横長バナーグラフィック）

### 加工方針

- **上記3のプロモーションバナーは完全に除去する。** 下端から順にスキャンし、
  彩度の高いマゼンタ〜ピンク系ピクセル（例: RGB (255,38,216) 付近）が
  出現し始める境界の少し上（安全マージンを含めてバナー全体が入らない位置）で
  画像をクロップする。目安としては画像下端から100〜160px程度の帯が対象。
  実際に生成した画像を確認し、バナーのグラフィックが1pxも残らないこと。
- **上記2のコピー文言「手で解け、口で蕩ける。」は残す。** ユーザーの指示により
  名刺にそのまま使う。クロップ時に文字が欠けないよう注意する。
- クロップ後の画像を、ページ全体（61mm×97mm、塗り足し込み・フチなし）に
  フルブリードで敷く。アスペクト比を保ったまま `object-fit: cover` 相当で
  配置し、キャラクターの顔・上半身が安全エリア内に収まるよう
  `object-position` を調整する。
- 背景として敷いた後、**画像下端（バナーを除去した境界のすぐ上）から
  下方向に向けて黒の半透明グラデーション（スクリム）を軽く重ねる。**
  これは次の「フッターバー」に自然につながるようにするための処理で、
  画像そのものを大きく暗くする必要はない（グラデーションの高さ目安:
  ページ下端から15〜20mm程度、不透明度は下に行くほど0→70%程度）。

### フッターバー（名前＋ロゴを乗せる新規要素）

画像下部（安全エリア内、ページ最下部寄り）に、名前と団体ロゴをまとめて
配置する帯状のエリアを新設する。

- 内容: 個人名「yuki__san」＋ `images/header-banner.png`（横長ロゴバナー、
  アスペクト比 5.36:1 を維持したまま幅24〜28mm程度で小さく配置）
- 配置順序: 名前を上、ロゴを下（またはその逆、視認性が良い方でよい）。
  名前が主役なので、ロゴより目立つ書体サイズ・ウェイトにすること
- 名前の文字スタイルは現行の名前プレート（`make_name_plate`）と
  同じ質感・エンボス調である必要はない。写真の上に直接乗せるため、
  白またはオフホワイト文字＋ドロップシャドウ（もしくは上記スクリムとの
  組み合わせ）で、暗い背景に対して十分なコントラストを確保すること
- 高さの目安: 安全エリア内、ページ下端から20〜24mm程度。名前・ロゴ・
  周囲の余白がこの範囲に収まるようバランスを取る
- ロゴ画像はCMYK入稿PDFでも使用するため、既存の
  `assets/header-banner-cmyk.jpg` 生成処理（`build.py` の `build_background()`）
  はそのまま使い続けてよい

### 装飾フレームについて

現行の箔押し風金枠・四隅の菱形アクセント（`draw_frame()`）は表面にも
引き続き使用する。写真の上に重ねても違和感が出ないか実際に生成して確認し、
枠線が細すぎて写真に埋もれる場合は不透明度や太さを調整してよい。
麻の葉模様（`draw_kumiko`）は表面では従来通り使用しない（写真と競合するため）。

## 裏面（back）のレイアウト変更

内容（団体ロゴ・団体名・mesukemo.uk・X・QR）は現行のまま変更しないが、
縦型（61mm×97mm、安全エリア49mm×85mm）に合わせてレイアウトを
横並びから縦積みに組み直す。

- ロゴ（`images/card-logo.png` の丸印）: 安全エリア上部、**水平方向中央**に配置。
  現行の20mm角から、幅に余裕があるため24mm角程度に拡大してよい
- 団体名「メスケモ推進委員会」: ロゴの下、水平方向中央揃え
- 副題「mesukemo.uk」: 団体名の下、中央揃え（現行の金色トーンを踏襲）
- X「@mesukemo_ya」: 副題の下（または横に並べず改行して）、中央揃え
- QR（`https://mesukemo.uk`）: 安全エリア下部、水平方向中央。
  縦方向の余裕があるため現行22mmから26〜28mm程度に拡大してよい
  （読み取り安定性が上がる）
- 背景の質感（実写木目・cedar系カラーグレーディング・箔押し枠・麻の葉の縁取り帯）は
  裏面は現行のまま踏襲する。縦横比が変わるため `mirror_tile` のタイリングが
  縦長になる点だけ確認すること

## 実装方針（ファイル別）

### `generate_background.py`
- `PAGE_MM = (61.0, 97.0)` に変更（DPI・WIDTH/HEIGHT は式で自動追従する）
- 表面の背景生成を、現行の「木目テクスチャ＋黒寄りカラーグレーディング＋
  名前プレート合成」から「`front-photo-source.png` を読み込み→バナー除去
  クロップ→フルブリード配置→下部スクリム→フレーム」に置き換える新関数を作る
  （例: `build_front_from_photo()`）。現行の `make_name_plate` を使った
  木札プレート合成コードは表面では使わなくなるため削除してよい（裏面や
  将来の再利用のために関数自体は残しても構わない）
- フッターバーの名前・ロゴはCMYK直描画PDF側（`build.py`）でテキスト／画像として
  重ねる方式でよい（現行の名前描画と同じ方式を踏襲）。背景PNG側にバナーの
  スクリムまでは焼き込み、名前文字はテキストとして重ねるのが望ましい
  （フォント埋め込み・検索性のため）
- 裏面の `make_common_background` 呼び出し・ロゴ合成・`draw_frame` は
  そのまま流用し、座標だけ新しい縦型サイズに合わせて中央寄せに変更する

### `card.html` / `card.css`（WeasyPrint, RGB確認用）
- `@page { size: 61mm 97mm; margin: 0; }` に変更
- `.card` の width/height を `61mm × 97mm` に変更
- 表面: `.character-photo`（個別画像タグでの合成）は廃止し、背景画像
  （`card-bg-front.png`、フルブリードの写真加工済み背景）1枚に一本化する。
  その上に `.name`（フッターバー内、テキスト）と `.banner-logo`
  （`images/header-banner.png`）を重ねる
- 裏面: `.identity` `.links` `.qr` を中央揃えの縦積みレイアウトに組み替える

### `build.py`（ReportLab, CMYK入稿用）
- `PAGE_MM` / `FINISH_MM` / `SAFE_MM` を新しい縦型の値に変更
- `write_cmyk_pdf_direct()` の表面描画部分を、現行の
  「名前バンド左上＋キャラ写真右＋バナー左下」の座標指定から、
  「背景画像フルブリード1枚＋フッターバー内に名前テキスト＋ロゴ画像」の
  座標指定に書き換える
- 裏面描画部分の座標を、縦積み・中央揃えの新レイアウトに合わせて書き換える
- `check_page_size()` の期待値（97×61mm）を新しいページサイズに合わせて修正

## セルフチェック

- `python build.py` が2ページのPDF（RGB・CMYK双方）を、縦型 61mm×97mmで出力すること
- 表面: プロモーションバナー（ピンク帯・マスコット・「メスケモ好きなら今すぐ入ろう」）が
  1pxも残っていないこと
- 表面: コピー文言「手で解け、口で蕩ける。」が欠けずに読めること
- 表面: 「yuki__san」の名前が写真の上で十分なコントラストを保ち、はっきり読めること
- 表面: `header-banner.png` のロゴが縮小後も文字がつぶれず判読できること
- 表面: 装飾フレーム（金枠・四隅の菱形アクセント）が写真の上でも視認できること
- 裏面: ロゴ・団体名・副題・X・QRがすべて水平中央揃えで縦に整列していること
- 裏面: QRが `https://mesukemo.uk` として正しくスキャン・デコードできること
- 全要素が安全エリア（49mm×85mm、各辺6mm内側）に収まっていること
- CMYK PDF内の画像がすべて `/DeviceCMYK` であること
- `print-card/README.md` に「縦型名刺」での入稿である旨の注意書きが追記されていること
