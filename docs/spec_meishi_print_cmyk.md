# 名刺データ作成仕様書（メスケモ推進委員会・アクセア入稿用）

## ゴール

リポジトリ内に新規フォルダ `print-card/` を作成し、アクセア（オンデマンド印刷）の定形名刺へ
そのまま入稿できる「完全データ」PDFを生成する仕組みを実装する。
Illustrator / Photoshop は使わない前提。再生成できるようスクリプト化し、READMEも付けること。

既存サイトのファイル（index.html / style.css / script.js / secret-card.html など）には
一切手を加えないこと。今回の成果物はすべて `print-card/` 配下に閉じて作成する。

## 印刷仕様（厳守）

- 仕上がりサイズ: 91mm × 55mm（横向き・定形名刺）
- 塗り足し: 各辺3mm。したがってPDFのページサイズは 97mm × 61mm
- トンボ（トリムマーク）は付けない。アクセアは「正寸＋塗り足しがあればトンボ無しで可」。
  ページサイズを 97×61mm にすること自体で塗り足しを表現する
- 断裁位置は各辺3mmの位置（＝仕上がり 91×55 の枠）
- 安全エリア: 各辺6mm内側（＝中央 85mm×49mm）。文字・ロゴ・QRなど重要要素は全部この中に収める
- 背景・地色はページの端(0mm)まで敷き切る（白フチが出ないように）
- 出力: PDF・1ページ・単位はmmで正確に。フォントは埋め込む（可能なら併せてアウトライン版も）
- カラー: CMYK

## 技術方針（Adobe不使用）

1. HTML + CSS でカードを作る。CSSの `@page` でページを正確に 97mm×61mm・余白0 に指定
2. WeasyPrint で HTML→PDF に書き出す（mm指定・フォント埋め込みができるため）
3. 書き出したPDFを Ghostscript で CMYK に変換。可能なら Japan Color 2001 Coated のICCプロファイルを適用
4. 確認用に RGB版PDF / CMYK版PDF / PNGプレビュー（原寸相当）を `print-card/output/` に出す
5. 下記フォントはプロジェクトに用意 or インストールしてから書き出す

## デザイン（既存サイトのトークンを流用）

- 世界観: 和風・木札風・金アクセント（サイトと統一）
- カラー: 木目茶 `#5c4033` / 金 `#d4af37` / 木目背景 `#8b6f47` / オフホワイト `#f5f3ed`
  - 注意: 金 `#d4af37` はCMYK変換でくすむ（金属光沢は4色印刷では再現不可）。
    それ前提の色設計にし、その旨コメントを残すこと
- フォント: 見出し Noto Serif JP / 本文 Zen Maru Gothic（どちらもGoogle Fonts・SIL OFLで印刷可）
- ロゴ: `images/card-logo.png`（410×409px, RGBA）を使用。
  - 名刺全面サイズ（97×61mm）を基準にした350dpi相当は約1337×841pxのため、
    そのままフルブリードで拡大使用すると解像度不足になる。
  - ロゴはヘッダー内の小さいマーク（目安：一辺20〜25mm程度以下）として配置し、
    その表示サイズにおける実効dpiを計算して350dpiを下回る場合はビルド時に警告を出すこと
  - 解像度計算式: `実効dpi = 元画像の一辺px ÷ (表示サイズmm ÷ 25.4)`
- QRコード: `https://mesukemo.uk` へのリンクを生成して配置。周囲のクワイエットゾーンを確保し、安全エリア内に置く
- レイアウトは片面・情報整理された名刺。要素は下の掲載内容を使う

## 最後にセルフチェック

- PDFのページサイズが正確に 97×61mm か
- フォントが埋め込まれているか
- 文字・ロゴ・QRが安全エリア（中央85×49mm）内に収まっているか
- 背景が端まで塗られ、白フチが出ていないか
- CMYKになっているか
- ロゴの実効dpiが350dpiを下回っていないか（下回る場合はREADMEに警告を明記）
