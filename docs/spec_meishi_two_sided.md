# 名刺を両面構成に変更（表: 名前+キャラ / 裏: 誘導系）

## 背景

現在 `print-card/` は片面（97mm×61mm、1ページPDF）で、団体ロゴ・団体名・
個人名・X・QRを1面に詰め込んでいる。これを両面構成に変更する。

- **表面**: 団体ロゴ（印）＋個人名＋キャラクターイラスト（今回はまだ画像が無いので
  スペースを空けておくのみ）。情報を絞ったシンプルな構成。
- **裏面**: 団体名「メスケモ推進委員会」＋ロゴ＋X＋QR＋mesukemo.uk（誘導系情報）。
  個人名は裏面には置かない。

出力PDFは1ファイル・2ページ（1ページ目=表、2ページ目=裏）とする
（日本の印刷所の両面入稿で一般的な形式）。ページサイズはこれまで通り
両ページとも97mm×61mm。

`print-card/` 以外のファイルには一切手を加えないこと。
背景の質感（実写木目のミラータイリング・箔押し風金枠・四隅の菱形アクセント・
金枠沿いの麻の葉帯）は表裏とも同じスタイルを踏襲すること
（`generate_background.py` の既存関数を再利用・共通化してよい）。

## 表面（front）のレイアウト

構図は「名前=左上（主役）／キャラクター=右（引き立て役、視線を名前に集める）／
ロゴ=左下に小さく（署名的に締める）」という優先順位で組む。理由: 名刺の主役は
名前であるべきで、カラフルなキャラクターを左や大きく置くと視線を持っていかれ
主従が逆転してしまうため。Zパターン（左上→右→左下）の視線誘導に沿わせる。

- **個人名「yuki__san」**（左上）: 既存の黒塗り木札プレート（`make_name_plate`と
  同じ質感・彫刻風エンボス文字）を使う。表面の主役なので視認性を最優先にする。
  位置は安全エリア左上、目安 幅46mm×高さ15mm程度で見栄えよく配置すること。
- **団体ロゴ**（左下、小さめ）: `images/header-banner.png`
  （960×179px、青地にピンク縁取り・白文字「メスケモ推進委員会」＋キャラの
  横長バナー画像。サイトのメインビジュアルで実際に使われている団体の公式ロゴ）を
  そのまま使用する。名刺全体の和風テイストとは色味が異なるが、団体の公式ロゴ
  なので気にせずそのまま使ってよい。安全エリア左下に、文字がつぶれず読める
  最小限のサイズ（目安 幅28〜32mm、アスペクト比 5.36:1を維持）で小さく配置する。
  目立たせすぎないこと。
- **キャラクターイラスト用のスペース**（右側）: 今回は画像ファイルが無いため
  実装しない。名前プレート・ロゴ以外の右側〜中央のエリアは無地の背景（模様なし）
  のままにしておくこと。将来ここに透過PNGを合成する前提でレイアウトを組むこと。

### 表面の背景色調について

表面は「黒と金」に見える色調にする。裏面（既存の cedar/wood-brown 系の
茶色トーン）とは別に、表面専用の背景バリエーションを作ること。

- `generate_background.py` の木目カラーグレーディングを表面用に複製・調整し、
  ベース色をより黒に寄せる（目安: 黒褐色〜ほぼ黒 `#0d0a08`〜`#1a1310` 程度への
  乗算・オーバーレイブレンド。現行の `#4a3028` / `#8b6f47` 寄せの処理を、表面では
  もっと暗いトーンに変更する）
- 箔押し風の金枠・四隅の菱形アクセント・麻の葉の縁取り帯は表面にもそのまま使う
  （黒地に金が映えて相性がよいはず）
- 名前プレート（黒塗り木札）が背景と同化しすぎて輪郭を失わないよう、
  プレート端の金の縁取りラインやドロップシャドウでの分離を保つこと
  （既存の `make_name_plate` の縁取り・影処理を活かせば問題ないはず。
  実際に生成した画像を見て、プレートの境界が判別できるか確認すること）

## 裏面（back）のレイアウト

現行の表面レイアウトから「個人名プレート」を取り除いたものとほぼ同じ。

- ロゴ: 左上、20mm角、(6mm, 6mm)
- 団体名「メスケモ推進委員会」＋副題「mesukemo.uk」: 現行と同じ位置・サイズ・
  letter-spacing
- X「@mesukemo_ya」: 現行と同じ位置・サイズ（副題と高さを揃える）
- QR（`https://mesukemo.uk`）: 現行と同じ位置・サイズ（右下22mm、台座付き）
- 個人名プレートは描画しない。空いたスペースは無理に埋めず、背景（木目＋
  縁の麻の葉帯）のみでよい

## 実装方針

### `generate_background.py`
- 木目タイリング・ビネット・麻の葉の縁取り・箔押し枠・四隅の菱形アクセントの
  各処理は関数として共通利用しつつ、カラーグレーディングの目標色だけ表裏で
  切り替えられるようにする（例: `color_grade_wood(img, mode="front"|"back")` の
  ように引数化するか、`color_grade_wood_dark()` を別途用意する）
  - 裏面: 現行の cedar/wood-brown 系グレーディングをそのまま使う
  - 表面: 黒寄りのグレーディング（上記「表面の背景色調について」参照）
- 表面: 共通背景（黒寄り） + 名前プレート（大きめ、左上）を合成。
  ロゴは焼き込まず、`images/header-banner.png` を `card.html`/`build.py` 側で
  画像としてそのまま重ねる（背景生成スクリプトに含めなくてよい）
- 裏面: 共通背景（茶系、既存のまま） + ロゴ（`images/card-logo.png`、円形の印）を
  現行同様に焼き込む
- 出力ファイル名（例。分かりやすければ変更してよい）
  - `assets/card-bg-front.png` / `assets/card-bg-front-cmyk.jpg`
  - `assets/card-bg-back.png` / `assets/card-bg-back-cmyk.jpg`

### `card.html` / `card.css`（WeasyPrint, RGB確認用）
- `.card` を表面用・裏面用の2つに分け、`break-after: page`（またはWeasyPrint対応の
  ページ区切り指定）で2ページのPDFになるようにする
- 表面: 名前プレート（左上）+ `images/header-banner.png`（左下、幅28〜32mm、
  アスペクト比維持）。ロゴ以外に団体名・副題・X・QRは置かない
- 裏面: ロゴ（丸印、背景に焼き込み済み）+ 団体名 + 副題 + X + QR（現行の該当CSSを
  ほぼ流用。個人名プレートのCSSは裏面には適用しない）
- 両ページとも `@page { size: 97mm 61mm; margin: 0; }` を適用すること

### `build.py`（ReportLab, CMYK入稿用）
- `write_cmyk_pdf_direct()` を表裏2ページ描画するように変更する
  - 表面を描画 → `c.showPage()` → 裏面を描画 → `c.showPage()` → `c.save()`
- `images/header-banner.png` はCMYK入稿PDFに埋め込むため、CMYK版のJPEGを
  生成すること（既存の `qr-mesukemo-cmyk.jpg` 生成と同様に
  `Image.open(...).convert("CMYK").save(...)` でよい。保存先は
  `print-card/assets/header-banner-cmyk.jpg` など分かりやすい名前でよい）
- `check_page_size()` は2ページになることを前提に、各ページのサイズを
  チェックするよう修正する（`len(reader.pages) != 2` でエラーにする）
- `write_preview()` は表裏それぞれのプレビューPNGを出力するようにする
  - `output/meishi_mesukemo_preview_front.png`
  - `output/meishi_mesukemo_preview_back.png`
  - Ghostscriptが無い場合は既存のPyMuPDFフォールバックを両ページ分行う

## セルフチェック

- `python build.py` が2ページのPDF（RGB・CMYK双方）を出力すること
- 各ページが97×61mmであること
- 表面: 個人名（左上）・`header-banner.png`（左下、小さめ）のみで、団体名テキストや
  X・QRが写っていないこと。背景が黒×金のトーンになっていること
- 表面: `header-banner.png` の文字「メスケモ推進委員会」が縮小後も潰れず判読できること
- 表面: 名前プレートの輪郭が背景（黒寄り）に対して埋没せず判別できること
- 裏面: 団体名・ロゴ（丸印）・X・QRが写っており、個人名が無いこと。背景は既存の
  茶系トーンのままであること
- QRが `https://mesukemo.uk` として正しくスキャン・デコードできること
- CMYK PDF内の画像がすべて `/DeviceCMYK` であること
- 表裏で金枠・麻の葉帯などのフレーム意匠は共通しつつ、背景色調は表(黒×金)と
  裏(茶系)で意図的に異なっていること
